## Next.js

### Installation

```bash
npm install @sylphx/silk @sylphx/silk-nextjs @sylphx/babel-plugin-silk
```

### Setup

#### Webpack Mode (Recommended - Zero Codegen)

```javascript
// next.config.mjs
import { withSilk } from '@sylphx/silk-nextjs'

export default withSilk({
  // Your Next.js config
}, {
  srcDir: './app', // or './src'
  debug: true
})
```

```json
// .babelrc
{
  "presets": ["next/babel"],
  "plugins": ["@sylphx/babel-plugin-silk"]
}
```

```typescript
// app/layout.tsx
import 'silk.css' // Virtual module generated by Webpack plugin

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

#### Turbopack Mode (Requires CLI Codegen)

```bash
npm install @sylphx/silk @sylphx/silk-nextjs @sylphx/silk-cli @sylphx/babel-plugin-silk
```

```javascript
// next.config.mjs
import { withSilk } from '@sylphx/silk-nextjs'

export default withSilk({
  // Your Next.js config
}, {
  turbopack: true, // Force Turbopack mode
  srcDir: './app',
  debug: true
})
```

```json
// package.json
{
  "scripts": {
    "predev": "silk generate --src ./app --output ./silk.generated.css",
    "prebuild": "silk generate --src ./app --output ./silk.generated.css",
    "dev": "next dev --turbo",
    "build": "next build"
  }
}
```

```typescript
// app/layout.tsx
import '../silk.generated.css' // Physical file generated by CLI

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

### Usage

```typescript
// app/page.tsx
import { css } from '@sylphx/silk'

const styles = {
  container: css({
    display: 'flex',
    padding: '2rem',
    background: 'linear-gradient(135deg, #667eea 0%, #764ba2 100%)'
  }),
  title: css({
    fontSize: '2rem',
    fontWeight: 'bold',
    color: 'white'
  })
}

export default function Page() {
  return (
    <div className={styles.container}>
      <h1 className={styles.title}>Hello Silk!</h1>
    </div>
  )
}
```

### Webpack vs Turbopack

| Mode | Setup | Codegen | CSS Output |
|------|-------|---------|------------|
| **Webpack** | ‚úÖ Auto | ‚ùå None | Virtual module `silk.css` |
| **Turbopack** | ‚úÖ Auto | ‚úÖ Required | Physical file `silk.generated.css` |

**Recommendation**: Use Webpack for true zero-codegen experience. Use Turbopack only if you need faster builds and don't mind the CLI step.

```typescript
// app/layout.tsx
import '../src/silk.generated.css'
```

  return <div className={styles.app.className}>Vite + Silk ‚úÖ</div>
}

## Nuxt 3

### Installation

```bash
npm install @sylphx/silk @sylphx/silk-nuxt
```

### Setup

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@sylphx/silk-nuxt']
})
```

That's it! The module auto-imports `silk.css` for you.

### Usage

```vue
<!-- pages/index.vue -->
<script setup lang="ts">
import { createStyleSystem } from '@sylphx/silk'

const { css } = createStyleSystem({})

const styles = {
  hero: css({
    display: 'flex',
    flexDirection: 'column',
    alignItems: 'center',
    padding: '4rem'
  } as any),
  title: css({
    fontSize: '3rem',
    fontWeight: 'bold',
    color: '#00DC82'
  } as any)
}
</script>

<template>
  <div :class="styles.hero.className">
    <h1 :class="styles.title.className">Nuxt + Silk ‚úÖ</h1>
  </div>
</template>
```

### Configuration (Optional)

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@sylphx/silk-nuxt'],

  silk: {
    srcDir: './app',      // Custom source directory
    debug: true,          // Enable debug logging
    minify: true,         // Force minification
    autoImport: true      // Auto-import silk.css (default: true)
  }
})
```

## üéØ Common Patterns

### TypeScript Type Assertions

All frameworks require `as any` for style objects due to strict CSS type checking:

```typescript
const styles = {
  button: css({
    padding: '1rem 2rem',
    backgroundColor: '#3b82f6'
  } as any)  // ‚Üê Required for now
}
```

We're working on improving type definitions to remove this requirement.

### Component Pattern

```typescript
import { createStyleSystem } from '@sylphx/silk'

const { css } = createStyleSystem({})

const Button = ({ children }) => {
  const styles = {
    button: css({
      padding: '0.5rem 1rem',
      borderRadius: '0.25rem',
      backgroundColor: '#3b82f6',
      color: 'white',
      _hover: {
        backgroundColor: '#2563eb'
      }
    } as any)
  }

  return <button className={styles.button.className}>{children}</button>
}
```
